IR Playbooks

INCIDENT RESPONSE — OT-Specific • NIST 800-82 Aligned • 4 Core Playbooks

OT-specific incident response playbooks designed for industrial environments where safety and process continuity are paramount. Each playbook follows a structured workflow from detection through recovery, with OT-safe containment procedures.

Playbook 1: Unauthorized PLC Access

  • Detect: IDS alert on unauthorized PLC connection or credential abuse
  • Isolate: Network-level containment without disrupting running process
  • Preserve Evidence: Capture network traffic, PLC logs, and memory dumps
  • Verify Logic Integrity: Compare running PLC logic against known-good baseline
  • Restore: Reload verified logic, rotate credentials, update firewall rules

Playbook 2: Process Anomaly

  • Alert Triage: Correlate IDS/SIEM alerts with process historian data
  • Process Verification: Validate physical process values against expected ranges
  • Root Cause Analysis: Determine if anomaly is cyber, mechanical, or operator error
  • Containment: Apply process-safe containment to affected zone
  • Recovery: Restore normal operations with enhanced monitoring

Playbook 3: Ransomware in OT

  • Network Segmentation: Immediately enforce IT/OT boundary and isolate affected segments
  • OT Impact Assessment: Determine which PLCs, HMIs, and historians are affected
  • Safe Shutdown: Execute controlled process shutdown if integrity is compromised
  • Air-Gap Recovery: Restore from offline backups with verified clean media

Playbook 4: Supply Chain Compromise

  • Firmware Verification: Hash-check all firmware and software against vendor-signed originals
  • Network Isolation: Quarantine devices with suspected compromised firmware
  • Behavioral Analysis: Monitor quarantined devices for C2 callbacks or anomalous traffic
  • Vendor Notification: Coordinate disclosure and remediation with equipment vendor
